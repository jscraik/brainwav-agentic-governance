# Glossary & Terminology

## Core Concepts

- **ArcTDD** – Arc-based Test-Driven Development; vertical slices of 2–5 commits within 120 minutes.
- **Phase Machine** – R→G→F→REVIEW workflow states (Red, Green, Fix/Refactor, Review).
- **Evidence Triplet** – Mandatory proof: milestone test red→green, contract snapshot, reviewer JSON.
- **Vibe Check** – Cortex Aegis oversight gate before side-effecting actions.
- **Run Manifest** – Machine log of tier, arcs, resets, evidence pointers at `tasks/<slug>/json/run-manifest.json`.

## Protocols & Standards

- **MCP** – Model Context Protocol; standardized tool integration for AI agents.
- **A2A** – Agent-to-agent event channel (CloudEvents-based).
- **SBOM** – Software Bill of Materials (CycloneDX 1.6+).
- **SLSA** – Supply-chain Levels for Software Artifacts (provenance attestation).

## Governance Terms

- **G0–G10** – Gate numbers in the agentic coding workflow (Intake through Archive).
- **Tier** – Task complexity: Tier 1 (fix), Tier 2 (feature), Tier 3 (research).
- **Waiver** – Approved exception to a governance rule, time-boxed and documented.
- **Preflight Guards** – Checks before action: academic research, license validation, connector health, vibe check.

## Security References

- **OWASP Top 10** – Web application security risks (2025 RC1).
- **OWASP ASVS** – Application Security Verification Standard (5.0.0).
- **OWASP LLM Top 10** – Large Language Model security risks (2025).
- **MITRE ATLAS** – Adversarial Threat Landscape for AI Systems.
